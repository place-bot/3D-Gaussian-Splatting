# GDC 2026 竞赛题目二：基于手机视频的室内高精度重建数据集说明

## 一、数据集简介

本数据集面向 GDC 2026 竞赛题目二“基于手机视频的室内高精度重建”，用于评估参赛算法在普通手机采集条件下，对真实室内场景进行三维重建的完整性、几何精度与工程可用性。

本赛项聚焦旧改施工、室内勘测等实际应用场景。参赛者需要基于给定的手机视频抽帧图像(初赛为图像帧，决赛为视频)，结合 SfM、MVS、NeRF、3D Gaussian Splatting 或其他三维视觉/几何重建方法，生成可用于施工勘测和尺寸量测的三维场景模型。

数据集覆盖不同规模、不同复杂度的室内环境，包含多房间、弱纹理等典型挑战场景。部分序列提供 FARO 等高精度扫描设备采集的参考点云，用于几何精度评估。

## 二、核心任务

参赛队伍需针对每个测试序列完成如下任务：

1. 输入 `images/` 目录中的连续图像帧。
2. 重建完整的室内三维场景模型。
3. 输出可量测、可查看、可评估的三维结果。
4. 在技术文档中说明重建流程、参数设置、硬件要求、精度评估方法与评估结果。
5. 提交演示程序及使用说明，评委可通过演示程序对测试数据集进行三维重建操作。

建议输出形式包括但不限于：

- 点云模型：`.ply`、`.pcd`、`.las`、`.laz`
- 三角网格模型：`.obj`、`.ply`、`.glb`、`.gltf`
- 相机位姿与内参：`.txt`、`.json`、COLMAP 格式文件等
- 精度评估报告：`.pdf`、`.docx` 或 `.md`

## 五、数据解析原则

1. 图像序列按文件名升序读取，视为原始手机视频的时间顺序。
2. 数据集中不提供相机内参、位姿或深度图。参赛者可自行完成相机标定、特征匹配、位姿估计、稠密重建与尺度恢复。
3. 若算法需要去模糊、动态物体过滤、弱纹理增强、闭环检测或多房间分块融合，应在技术方案文档中明确说明。
4. 输出模型需与参考真值处于同一尺度。若算法存在尺度不确定性，应说明尺度对齐方法。
5. 对于多房间和弱纹理序列，应重点关注全局一致性、房间连接关系、墙面/地面完整性以及局部缺失区域处理。

## 七、评测指标

竞赛重点考察模型完整性、几何精度、视觉质量与工程可用性。

### 1. 几何精度

以重建模型与参考真值点云之间的空间距离为主要依据。基础目标为模型精度达到 20 cm；达到 10 cm 以内可作为加分项。

建议报告以下指标：

- 平均距离误差
- 中位数距离误差
- RMSE
- 90% / 95% 分位误差
- 20 cm 内点比例
- 10 cm 内点比例

### 2. 场景完整性

评估室内主要结构是否完整重建，包括墙面、地面、顶面、门洞、转角、房间连接区域以及大件固定结构。多房间场景需关注跨房间尺度一致性与拓扑连贯性。

### 3. 模型质量

评估模型表面噪声、孔洞、漂浮点、重影、局部扭曲、纹理错位和边界破碎等问题。对于弱纹理场景，应重点说明算法如何处理白墙、重复纹理、反光或曝光变化。

### 4. 工程可用性

评估演示程序的可运行性、处理效率、硬件门槛、参数易用性与结果导出能力。提交的软件应尽量便于评委在统一环境下复现。

## 八、推荐评估流程

1. 对重建结果与 `gt/gt_pd.ply` 进行对齐。
2. 计算重建结果到参考点云的最近邻距离。
3. 统计误差分布，并生成误差热力图或彩色点云。
4. 在报告中给出各序列的量化指标、可视化结果与失败案例分析。

常见工具包括 CloudCompare、Open3D、MeshLab、COLMAP、Blender 以及自研评估程序。参赛队伍可自行选择工具，但需保证评估方法清晰、可复核。

## 九、注意事项

- 请勿在重建阶段使用 `gt/gt_pd.ply` 作为输入。
- 若使用预训练模型、开源工程或第三方库，应在技术文档中列明来源与版本。
- 若对图像进行了筛选、降采样、增强或去动态物体处理，应记录处理策略。
- 若结果文件体积较大，建议同时提交轻量化预览模型和完整精度模型。
- 最终提交文件命名应清晰，保证每个结果可追溯到对应序列。
